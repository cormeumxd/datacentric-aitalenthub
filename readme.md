# Документация скиллов проекта opencode

Проект использует модульную архитектуру скиллов для автоматизации полного цикла создания ML-решений по работе с новостными данными. Координатором всех процессов выступает **Main Skill**.

## Основные компоненты

### 1. Main Skill
Центральный скилл, управляющий всем жизненным циклом проекта: от сбора данных до развертывания модели.
- **Функция:** Оркестрация этапов, обеспечение передачи данных между компонентами, контроль качества.
- **Рабочий процесс:**
  1. Вызывает *Data Collection* для парсинга новостей.
  2. Запускает *Data Quality* для оценки и очистки набора данных.
  3. Использует *Data Labeling* для разметки (включая интеграцию с LabelStudio).
  4. Применяет *Active Learning* для эффективного обучения модели.
  5. Формирует финальный отчет о результатах.

---

### 2. Сопутствующие скиллы

*   **Data Collection Skill**
    *   **Назначение:** Автоматизированный поиск и сбор новостных датасетов из Интернета и библиотек (например, Hugging Face).
    *   **Роль:** Первичная поставка «сырых» данных для обработки.

*   **Data Quality Skill**
    *   **Назначение:** Анализ качества данных.
    *   **Роль:** Выявление аномалий, дублей, пропусков и предоставление стратегий по исправлению для улучшения чистоты датасета.

*   **Data Labeling Skill (AutoLabel)**
    *   **Назначение:** Автоматическая разметка данных.
    *   **Роль:** Генерация спецификаций разметки, оценка уверенности модели и экспорт данных в LabelStudio для ручной верификации человеком.

*   **Active Learning Skill**
    *   **Назначение:** Итеративное обучение и отбор данных.
    *   **Роль:** Интеллектуальный выбор наиболее значимых примеров для дообучения модели, что минимизирует объем необходимой ручной разметки.

---

## Использование
Для начала работы вызовите **Main Skill** через интерфейс агента. Он будет последовательно запрашивать у вас подтверждение для перехода между этапами, обеспечивая режим *Human-in-the-loop*.
